10 quick tests to measure your site’s accessibility

Toufic Sbeiti, Manager of Web Operations at Citizenship and Immigration Canada, will present 10 tests that can be performed quickly to measure the accessibility of a web page. If you are a developer, you can fix issues before posting live. If you are a project manager or a client, you can ensure the level of accessibility without advanced coding knowledge.

What to bring: Bring a laptop (FireFox with Wave and Web Developer toolbars) and follow the steps to test your web page during the presentation.

Toufic Sbeiti

Manager - Web Operations, Citizenship and Immigration Canada

Audience: Developers

Level: Beginner

Language: English

ATAG & Drupal 8

The Future of Accessible Content is going to be ATAG 2.0. A standard that both ensures that the editing interface is accessible as well as that there are measures to see that content that is produced with it is accessible. Although Drupal 8 isn't out, and ATAG 2.0 is still in draft, looking ahead it is important to anticipate what impact this will have for government sites. We can highlight enhancements to the admin side of Drupal 8, talk about the current Web Experience Toolkit Drupal implementation and what it has already adopted from Drupal 8's approach.

What to bring: Bring your laptop with a modern web browser installed.

OpenConcept Consulting Inc.

Content inventory for website redesign

Whether you’re migrating content to a content management system, working with writers to renovate and re-write content, or designing and information architecture, a content inventory is a tool that you will return to again and again. This is especially true when it comes to redesign projects that involve content-heavy websites. In this session, we will work together to identify best practices and lessons learned to help establish the content inventory framework for the Government of Canada’s Web Renewal Initiative.

GitHub 101: Setting up and navigating

Afraid of GitHub? Intimidated by your GitHub-savvy colleagues? Maybe there's a good reason for that. But in this session we'll work through those fears together and get comfortable navigating the GitHub.com interface. Starting with setting up your own GitHub account, we'll follow other users, watch some projects, join an organization and get to know the GitHub Octocat. No command line, no Git, no deep principles of distributed version control. This session is for the true GitHub beginner - but by the end of this short session you'll have faced your GitHub fears and be ready to interact with your designer and developer colleagues.

What to bring: An Internet-enabled device (preferably a laptop) with a web browser.

Justin Longo

PostDoc in Open Governance

Audience: Anyone who hasn't used GitHub at all (or much) but should.

Level: Beginner

Language: English

GitHub 102: Fork this repo!

So you've lurked around the edges of GitHub for a while but your profile page still shows "0 Contributions". Well, it's time to put some green blocks on that screen. Building on the previous session (GitHub 101: Setting up and navigating), this session will focus on making contributions through GitHub.com and interacting with other users. You'll create your first project (called a repository, or repo), “fork a repo”, flag an issue, issue a pull request, and manage pull requests from other users. We'll do all of this and not even peak at the command line. This session is for the true GitHub beginner - but by the end of this short session you'll have earned some props from your designer and developer colleagues and move towards being a regular GitHub contributor.

What to bring: An Internet-enabled device (preferably a laptop) with a web browser. Assumes completion of the session "GitHub 101: Setting up and navigating"; or an account on GitHub.com and familiarity with the interface.

Justin Longo

How to do online user experience research with Optimal Workshop

If you thought you didn't have the capacity to do some user research in-house, think again! During this session, you will learn how to do online card sorting, as well as navigation and prototype testing. We will go over the basics of each of these user research methods and then explore how you can set them up using the Optimal Workshop suite of tools: OptimalSort, Treejack and Chalkmark. The presentation will explore when to use each technique/tool and how to analyze the results with real examples from the Canada.ca website and from other departmental projects. The session will be co-presented by Peter Smith, Lead, Product Development for Canada.ca, Treasury Board Secretariat.

How to migrate to Web Experience Toolkit version 4

Join us for a discussion on the migration process and, more broadly, impacts and decisions required for content development, organization and management.

Web accessibility testing using NVDA

Accessibility testing using browser tools such as the Web Developer or the WAVE toolbars is already a daunting task; add screen reader testing requirements to the mix and for most people, it soon becomes way too complicated to even bother. This session is intended for anyone who does not consider himself or herself to be an accessibility expert, but yet needs to sometimes access accessibility. In this session, we will be going over a series of techniques for testing the accessibility fundamentals of HTML using NVDA, an open source screen reader. No technical ninja skills required! Join us if you are willing to discover 12 simple technique in 30 quick minutes!

What to bring: Your worst fears and concerns about the use of screen readers for accessibility testing, so we can crush them to a pulp.
