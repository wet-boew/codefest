Code sprints

What is a code sprint?

A code sprint is a get-together of people involved in a project to give a focused development on the project.

Leading a code sprint

Leading a code sprint at CodeFest is simple:

  1. Give a 10‒15-minute introduction explaining what you're hoping to accomplish during the sprint.
  2. Split participants into groups and divide up the work. If a task is complex, you should do a dry-run with a small portion of the work to make sure that everyone is on the right track.
  3. Code, help participants and answer questions.

Participating in a code sprint

Participating in a code sprint involves coding, so you will need to bring a computer.

You will also need to following software installed on your computer:

  • A Web browser with debugging tools, such as:
    • Firefox with the Firebug add-on.
    • Google Chrome.
    • Safari.
    • Internet Explorer 10.
  • A text editor or intregrated development environment, such as:
    • Notepad++.
    • Aptana.
    • Eclipse.
    • Netbeans.
  • A git client.
  • Node.js.
  • Grunt.

We recommend that you install everything before the event. But, if you have any trouble setting up your computer, we will have people to help you on-site.

Tabbed interface redesign

This sprint is for everyone that wants to improve the Web Experience Toolkit's tabbed interface. We'll be working on new, responsive tab panel designs, replacing crufty JavaScript with sleek CSS3 and getting the plugin ready for version 4.0 of the Toolkit.

  • Responsive Web design
  • CSS-only tabs

Extend charts and graphs

Use and add Flot plugin in the charts and graphs plugin of the Web Experience Toolkit. Data source represented in a HTML table. Use CSS parameter/options, HTML data-* attibute and JSON parameter/options.

  • Presentation for the extend charts and graphs code sprint
  • Flot
  • Flot examples
  • Flot plugins
  • Charts and graphs plugin

Apache Cordova (Phonegap) variant of the Web Experience Toolkit for native mobile applications

The goal is to develop and maintain a Cordova codebase for native mobile/tablet applications using the Web Experience Toolkit as the base Web framework. The objective is to add cross-platform functionality for camera, local storage, push notifications, and other device features currently not supported by HTML5.

  • Android SDK
  • Node.js
  • Apache Cordova
